# step-01-init: Opal Requirements Discovery

## MANDATORY EXECUTION RULES

1. **Use Cases First:** Document AI use cases before designing agents.
2. **Pain Points Drive Value:** Prioritize use cases that solve actual editorial pain points.
3. **Credit Budget Awareness:** Plan agents within allocated credit budget (baseline 200/month).
4. **Team Involvement:** Involve editorial team in use case identification.

## YOUR TASK

### A. Identify AI Use Cases

Assess editorial workflows and identify opportunities for AI augmentation:

- **Content Generation:** Generate draft articles, social media posts, product descriptions
- **Translation:** Translate content to multiple languages
- **SEO Optimization:** Improve titles, descriptions, meta tags for search
- **Personalization:** Generate personalized content variants based on audience
- **Workflow Automation:** Automate approval routing, publication scheduling
- **Accessibility:** Auto-generate alt text, captions for video

### B. Assess Current Workflows

Document how editorial team currently works and where AI could add value:

- Time spent on repetitive tasks (manual writing, translation, optimization)
- Error rates in current workflows (typos, SEO gaps, consistency issues)
- Team skill gaps (limited translation, technical SEO expertise)
- Bottlenecks in approval/publishing

### C. Document Requirements

Create Opal requirements document with:

- Identified use cases (5+ scenarios)
- Priority ranking (high/medium/low based on pain, impact, effort)
- Credit budget allocation per use case
- RAG knowledge sources (existing brand guidelines, style guides)
- Team training needs

### D. Plan Opal Configuration

Document:

- Agent types needed (general assistant, specialized agents)
- Tool requirements (CMS connectors, custom tools via SDK)
- Knowledge sources (documents for RAG)
- Success metrics per agent

## SUCCESS METRICS

- [ ] Use cases identified (5+)
- [ ] Workflows documented
- [ ] Pain points quantified
- [ ] Credit budget planned
- [ ] Team input gathered
- [ ] Requirements document signed off
