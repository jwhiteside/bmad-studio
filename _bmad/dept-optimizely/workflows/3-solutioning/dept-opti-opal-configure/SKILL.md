# dept-opti-opal-configure

Configure Optimizely Opal AI agents and workflows for content generation, translation, personalisation, and workflow automation.

## Purpose
Design and deploy AI agents leveraging Opal AI capabilities to augment editorial workflows, content creation, and personalization within an Optimizely implementation.

## Phase
Phase 3: Solutioning

## Scope
- Identify AI use cases for the programme (content generation, SEO, translation, personalisation)
- Design agent architectures and tool integrations
- Configure RAG (Retrieval-Augmented Generation) knowledge sources
- Design agent instructions (personal + org-wide)
- Implement custom tools via Opal Tools SDK if needed
- Set up evaluations framework
- Plan deployment and credit management

## Key Principle
**AI Augments, Not Replaces.** Agents support editorial team workflows; content specialists retain approval authority and editorial judgment.

## Deliverables
- Opal use case assessment and roadmap
- Agent design specifications (purpose, tools, instructions, RAG sources)
- Evaluation framework (test cases, success metrics)
- Deployed agents (Agent Directory entries)
- Operational runbook (credit tracking, monitoring, escalation)
- Training materials for editorial team

## Success Metrics
- Use cases identified and prioritized
- Agents deployed and tested
- Evaluation pass rate >= 80%
- Credit usage tracked and optimized
- Editorial team trained and adopting agents
- Agent Directory entries published

## Platform
SaaS CMS only (Opal is SaaS feature)

## Duration
3–5 days (design + implementation + evaluation)

## Inputs
- Current content workflows
- Optimizely Opal subscription (with credit allocation)
- Content examples for RAG training
- Editorial team feedback on pain points

## Outputs
- dept-opti-opal-configure workflow execution
- Deployed agents in Agent Directory
- Approval/Preferred/Caution menu for proceed/iterate

## Related Workflows
- dept-opti-build-component (components for agent outputs)
- dept-opti-migration-execute (agents support content migration)

## Next Steps
After approval, proceed to Phase 4 implementation and deployment.
