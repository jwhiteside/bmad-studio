# dept-opti-cmp-setup

Configure Optimizely CMP (Campaign Management Platform) for editorial workflows, approval routing, and publishing.

## Purpose
Set up Optimizely CMP to support editorial team workflows, campaign planning, approval routing, and integration with CMS publishing.

## Phase
Phase 4: Implementation

## Scope
- Define campaign types and task workflows
- Configure editorial calendars and scheduling
- Set up approval routing rules and escalation
- Integrate CMP with CMS publishing (CMS 12 or SaaS CMS)
- Configure analytics and reporting
- Set up user roles and permissions
- Train editorial team on CMP workflows

## Key Principle
**Workflow Mirrors Reality.** CMP configuration should match how editorial team actually works, not how they think they should work.

## Deliverables
- CMP configuration (templates, workflows, approvals)
- Integration setup (CMS publishing connections)
- Editorial calendar configuration
- User roles and permissions mapping
- Training materials and runbook
- Go-live checklist

## Success Metrics
- Campaign types configured per requirements
- Approval workflows tested and working
- CMS integration validated
- Editorial team trained and adopting
- Reporting working (campaign performance metrics)

## Duration
2–3 days (setup + training + validation)

## Inputs
- Editorial team structure and workflow requirements
- CMS platform (CMS 12 or SaaS CMS) integration details
- Campaign types and approval rules
- Reporting requirements

## Outputs
- CMP instance configured and tested
- Integration verified
- Training completed
- Approval/Preferred/Caution menu

## Next Steps
After go-live, provide operational support (30 days) and then transition to BAU team.
