# dept-opti-cmp-setup Workflow

**Workflow Name:** Configure Optimizely CMP for Editorial Workflows
**Phase:** Phase 4 — Implementation
**Duration:** 2–3 days

## Workflow Overview

This workflow guides you through setting up Optimizely CMP (Campaign Management Platform) to support editorial team workflows, approval routing, and CMS publishing integration.

## Key Principle

**Workflow Mirrors Reality.** CMP configuration should match how editorial team actually works, not how they think they should work.

## Workflow Structure

```
step-01-init
    ↓ (workflows documented, team structure mapped)
step-02-configure
    ↓ (CMP configured, integrated, team trained)
DECISION GATE → [Go-Live Approved]
```

## Step Overview

- **Step 1:** Document campaign types and approval workflows, map editorial team structure to CMP roles, plan CMS integration points, create CMP configuration document
- **Step 2:** Configure campaign templates and task workflows, set up editorial calendars, configure approval routing rules, integrate with CMS publishing, train content team, document operational runbook

## Success Metrics

- Campaign types configured per requirements
- Approval workflows tested and working
- CMS integration validated
- Editorial team trained and adopting
- Reporting working (campaign performance metrics)

## Next Step

After go-live, provide operational support (30 days) and transition to BAU team.
