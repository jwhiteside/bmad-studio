# step-02-configure: CMP Configuration and Integration

## MANDATORY EXECUTION RULES

1. **Configuration Matches Requirements:** Each CMP setting should trace back to step-01 requirements.
2. **Integration Tested:** CMS integration verified before team training.
3. **Team Trained:** Editorial team completes training and successfully executes test campaign before go-live.

## YOUR TASK

### A. Configure Campaign Templates

For each campaign type from step-01:

- [ ] Create campaign template
- [ ] Define required fields (title, content, budget, channels)
- [ ] Set campaign duration default
- [ ] Configure task workflow (creation → review → approval → publishing)

### B. Configure Approval Routing

Map approval rules:

- [ ] Define approval hierarchy (who approves which campaigns?)
- [ ] Set escalation rules (time limits for approval?)
- [ ] Configure notifications (who gets notified?)

### C. Integrate with CMS

**For CMS 12:**

- [ ] Configure CMS content type integration
- [ ] Test CMP content creation → CMS draft
- [ ] Verify CMS publishing from CMP

**For SaaS CMS:**

- [ ] Configure REST API integration (auth, endpoint)
- [ ] Map CMP fields to CMS content type
- [ ] Test content creation and publishing

### D. Configure Editorial Calendar

- [ ] Create editorial calendar template
- [ ] Add campaign types as calendar events
- [ ] Share calendar with team
- [ ] Test calendar functionality

### E. Set Up Reporting

Configure analytics/reporting:

- [ ] Campaign performance dashboard (if applicable)
- [ ] Campaign status reports
- [ ] Approval timeline reports
- [ ] Publishing success rates

### F. Train Editorial Team

Training plan:

- [ ] Create campaign tutorial (step-by-step)
- [ ] Demonstrate approval workflow
- [ ] Show CMS integration
- [ ] Execute test campaign with team
- [ ] Collect feedback

### G. Create Operational Runbook

Document:

- [ ] How to create campaign
- [ ] How to submit for approval
- [ ] How to publish to CMS
- [ ] Troubleshooting (common issues + fixes)
- [ ] Escalation contacts

## SUCCESS METRICS

- [ ] Campaign templates configured per requirements
- [ ] Approval workflows tested
- [ ] CMS integration validated
- [ ] Team trained and comfortable
- [ ] Test campaign executed successfully
- [ ] Operational runbook documented

## NEXT STEP

→ After go-live, provide 30-day support and transition to BAU team
