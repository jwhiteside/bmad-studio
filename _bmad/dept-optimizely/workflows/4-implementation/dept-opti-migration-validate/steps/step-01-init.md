# step-01-init: Validation Setup

## YOUR TASK

### A. Load Migration Reports

- Load wave execution reports from dept-opti-migration-execute
- Review completion status per wave
- Document any exceptions or rework completed

### B. Define Validation Criteria

Define what "success" looks like:

- **Completeness:** % of content items successfully migrated (target >= 99%)
- **Quality:** Field mapping accuracy, no data loss (target >= 99%)
- **URLs:** Legacy URLs redirect to new URLs (target >= 99%)
- **SEO:** Meta tags, structured data preserved (target >= 95%)
- **Performance:** Page load times meet baseline (target < 3s)

### C. Set Up Validation Tracking

Create validation document with:

- [ ] Content inventory (legacy count vs. migrated count)
- [ ] Field mapping spot-check samples
- [ ] URL redirect testing sample (10-20 URLs)
- [ ] SEO metadata validation
- [ ] Performance baseline measurements
- [ ] Go-live checklist

## SUCCESS METRICS

- [ ] Migration reports reviewed
- [ ] Validation criteria documented
- [ ] Tracking document created
