# Step 2: Source Data Analysis

## Objective

Analyze source system data structure and available events/data.

## Instructions

For source system:
- Extract sample data showing all fields and variations
- Document data types, lengths, formats
- Identify calculated/derived fields
- Document any constraints (required fields, valid values, uniqueness)
- Identify events that trigger data updates (when does data change?)
- What is the current data quality? (completeness, accuracy)

Create source data specification table:

| Field | Type | Length | Required | Example | Notes |
|---|---|---|---|---|---|
| sku | String | 50 | Yes | ABC-123 | Unique product identifier |

## Outputs

- Source data specification
- Data quality assessment
- Sample data extract
