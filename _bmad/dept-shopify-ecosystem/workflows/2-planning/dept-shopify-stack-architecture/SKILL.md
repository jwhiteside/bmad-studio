# Shopify Stack Architecture Design

**Entry Point**: `dept-shopify-stack-architecture`

Design comprehensive technical architecture for Shopify ecosystem implementation. Selects integration architecture pattern, defines data ownership and customer identity strategy, maps all data flows between platforms, and documents architecture decisions.

## What This Skill Does

- Reviews ecosystem assessment and platform selections
- Selects integration architecture pattern (hub-spoke, middleware, event-driven, headless)
- Defines data ownership model and customer identity strategy
- Maps all data flows between platforms with transformation rules
- Creates architecture decision records for key technical decisions
- Produces architecture document suitable for implementation team

## When To Use It

- After platform selections are finalized (PIM, CRM, CDP, etc.)
- Before beginning integration implementation
- When defining technical governance and standards
- When planning data architecture and identity management

## Inputs

- Ecosystem assessment with recommended pattern
- Platform selections and vendor confirmations
- Business requirements and integration priorities
- Data governance and compliance requirements

## Outputs

- Architecture design document with:
  - Integration pattern selection and rationale
  - Data ownership matrix (which system owns which data)
  - Customer identity and resolution strategy
  - Complete data flow diagrams and specifications
  - Architecture decision records (ADRs)
  - Technical standards and constraints
