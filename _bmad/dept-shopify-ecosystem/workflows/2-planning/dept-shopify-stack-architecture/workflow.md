# Workflow: Shopify Stack Architecture Design

## Overview

Design technical architecture for Shopify ecosystem implementation. Defines integration patterns, data ownership, customer identity strategy, and data flows between all platforms.

## Prerequisites

- Ecosystem assessment completed with recommended pattern
- Platform selections finalized (Shopify Plus, PIM, CRM, CDP, etc.)
- Business requirements and integration priorities documented
- Data governance and compliance requirements understood

## Steps

1. **Review & Kickoff**: [step-01-init](steps/step-01-init.md)
2. **Integration Pattern Selection**: [step-02-integration-pattern](steps/step-02-integration-pattern.md)
3. **Data Architecture Design**: [step-03-data-architecture](steps/step-03-data-architecture.md)
4. **Data Flow Mapping**: [step-04-data-flows](steps/step-04-data-flows.md)
5. **Architecture Decision Records**: [step-05-adrs](steps/step-05-adrs.md)
6. **Architecture Documentation**: [step-06-document](steps/step-06-document.md)

## Completion Criteria

- Integration pattern selected with justification
- Data ownership matrix complete for all data entities
- Customer identity/resolution strategy defined
- All significant data flows mapped and specified
- Key architectural decisions documented in ADRs
- Architecture document ready for implementation team
- Enterprise Architecture and Data teams sign-off
